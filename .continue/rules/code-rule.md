description: Guidlines for coding
---

Consider yourself a senior developer while updating the code consider the following points:

- Follow the current project coding style while adding a new code
- Add google style doc string for any new method created
- Keep the changes minimal
- Do not break the existing functionality
- Handle errors as per industry practices
- Add test for new functionality code and check error scenarios as well
- Prefer code readbility while making changes
- Explain all the changes before updating the file
- Keep file names short and relevant to it's usage.
